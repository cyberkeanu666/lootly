Šta je Lootly?
Platforma za fer, transparentne i viralne giveaway kampanje

Lootly je web platforma koja omogućava kreatorima (YouTuberi, streameri, brendovi) da kreiraju giveaway kampanje, podele jedinstveni link i izvrše live izvlačenje pobednika — na način koji je kriptografski dokazivo fer, tako da niko ne može da tvrdi da je namješteno.

💡
Ključna ideja: Host kreira kampanju → generiše se unique link → učesnici se prijavljuju verifikacijom Instagram username-a → na dan izvlačenja, provably fair algoritam bira pobednika → host jednim klikom verifikuje da li pobednik prati sve zahtevane profile.

Verifikacija — Bio kod metoda
Bez Meta API approval — radi odmah
⚠️
Username je LOCKED samo nakon uspešne verifikacije. Ako neko unese tuđi username — taj pokušaj expira za 30 min bez ikakve štete za vlasnika profila. Vlasnik može normalno da uđe i verifikuje se sa istim username-om.
// Verifikacioni flow 1. Učesnik unese username → sistem generiše: "LOOTLY-X7K2-[giveaway_id]" // vezano za username + giveaway + timestamp 2. Kod važi 30 minuta, max 3 pokušaja po IP-u 3. Učesnik stavlja kod u Instagram BIO 4. Klikne "Verify Me" → job ide u BullMQ queue 5. Puppeteer otvara javni profil, čita bio text → Kod pronađen ✅ → tiket izdat, username LOCKED → Kod nije pronađen ❌ → retry (do 3x) 6. Atomic Firestore transaction za race condition zaštitu → 2 osobe pokušaju isti username istovremeno → Prva uspešna verifikacija dobija tiket, druga grešku
Username status pravila
🔴Unet, verifikacija pending → Slobodan
🔴Unet, verifikacija failed → Slobodan
🔴Unet, kod expired → Slobodan
🟢Verifikovan ✅ → LOCKED
Anti-abuse zaštita
→ Isti username = 1 tiket po giveaway-u
→ Isti IP = max 3 različita username-a/dan
→ Rate limit: max 10 scrape-ova/min (BullMQ)
→ Kod je hash — ne može se brute-force-ovati

Provably Fair Draw sistem
Kriptografski dokazivo fer — niko ne može da namesti
// Korak 1 — Kada se kreira giveaway const seed = crypto.randomBytes(32).toString('hex') const seedHash = crypto.createHash('sha256').update(seed).digest('hex') // seedHash se odmah objavljuje javno na stranici giveaway-a // seed ostaje tajan sve do dana izvlačenja // Korak 2 — Na dan izvlačenja // seed se objavljuje uživo — svako može da proveri: SHA256(revealed_seed) === published_hash ✅ // Korak 3 — Određivanje pobednika const sorted = participants.map(p => p.username).sort() const finalHash = SHA256(seed + sorted.join(',')) const winnerIndex = parseInt(finalHash, 16) % sorted.length // Deterministički, reproducibilno od strane bilo koga
🛡️
Hash je objavljen pre nego što se iko prijavio. Čak ni host nije mogao da zna ko će pobediti. Svako može samostalno da reprodukuje rezultat sa objavljenim podacima.

Rizici & napomene
Ovo treba znati pre nego što kreneš
🚨
Puppeteer & Instagram scraping: Instagram aktivno blokira scraping. Koristiti rotate user-agents, random delays, i RapidAPI fallback. Ovo je najveći tehnički rizik — testiraj PRVO pre ostalih faza.
⚡
Socket.io sa 10K+ korisnika: Jedan Node.js process ima limite. Za produkciju razmotriti Redis adapter za Socket.io i horizontalno skaliranje.
💡
Simple mode uvek dostupan: Host može pokrenuti giveaway bez ikakve verifikacije. Korisno za testiranje i za kreatore koji žele brz setup.
🔑
Meta API nije potreban: Ceo sistem radi bez Meta/Instagram API approval. Puppeteer scrape-uje javne profile. Jedina ograničenja su brzina i IP ban rizik — rešeno BullMQ queue-om.
